Why inspect hydrological data?

Before you begin any practical task in hydrology—building a model, determining a return period, designing a reservoir, etc.—it’s ususally a good idea to inspect your data. You want to make sure that the data are fit for purpose. Are there trends? What are the significant events? Are they gaps and can they be filled? What is the seasonality like? How can the data be manipulated to better suit the task at hand? This tutorial highlights some of the basic tools in R that can enable you to answer these questions quickly and easily. I use a monthly streamflow record as an example, but the same techniques can be applied to any hydrological variable (e.g., rainfall, temperature) at any resolution (hourly, daily, weekly, etc.).

For this tutorial you will need to download a monthly streamflow record for the River Gryfe, located here. Copy the file to your working directory. If you don’t know your working directory, simply type getwd() into the R consule.

Time series objects

In R, the in-built ‘stats’ package contains a selection of functions for creating and analysing time series objects. A time series (‘ts’) object is a sequence of data that have been sampled at equispaced points in time, such as hourly, daily, weekly, monthly, quarterly, annually, and so on. In R, you can create a ts object from any vector of equispaced samples, such as the ‘Q’ variable in the downloaded example record, by feeding the vector into the ts function. Two additional arguments are required to time-stamp the data: start and frequency. Intuitively start is used to tell the function the time of the first observation, and frequency is used to tell the function how many data points you have per unit of time1 For example, if you have hourly data, you can time-stamp it by assuming the unit of time is one day (frequency = 24). Alternatively, you could assume the unit of time is one week (frequency = 168), and so on.. In this example you have monthly data and you will assume a unit of time as one year (frequency = 12). Later you’ll see how having data described by two levels of time (month and year) can be extremely useful.

Convert the observed streamflow record for the Gryfe River into a time series object as follows:

Qts <- ts(data = Qdata[,2], start = c(1963, 7), frequency = 12)

Note that the plot automatically gives the time along the x-axis and that, when printed, the data are displayed as a matrix with months as column headers and years as row names. You can look at the data and quickly spot a number of important features. For example, there is a signicant gap in the record. As a hydrologist, you might look to establish the cause of missing data2 It turns out that the weir used for flow measurement collapsed in 1996 and wasn’t repaired till 2002 and whether there are rainfall records for the catchment that might enable allow you to reconstruct the record. It’s also clear from the consistent fluctuation of the series that the data are highly seasonal. If you want to take a closer look at the seasonal pattern, you can expand out the time series across a number of plots using the xyplot.ts function in the lattice package. You can further examine seasonality in your time series using monthplot, which plots the individual series for each month:

monthplot(Qts, ylab = "Streamflow rate [cumecs]")

The strong seasonal pattern is clear from this plot. If you’re interested in whether there’s any discerable trend or shift in the time series, you can remove the seasonality. The easiest way to do this is to aggregate to mean annual flow rate:

By setting FUN = mean you tell the function to aggregate each year of data by taking the mean across all months in each year4 We can substitute mean for max, min, sum, or indeed any function that can be applied to the data..

Step 3: Perform some quick statistical analysis

The data appear to exhibit a shift in the mean annual streamflow, occurring around 1980. You can use the window function to separate out the time series object for computing statistics relating to pre- and post- 1980:

Gryfe River mean annual streamflow 

plot(Qannual, ylab = "Streamflow rate [cumecs]")
Qannual_pre1980 <- window(Qannual, end = 1980)
Qannual_post1980 <- window(Qannual, start = 1981)

# PLOT MEAN OF EACH PERIOD FOR COMPARISON
abline(h=mean(Qannual_pre1980), col = "blue")
abline(h=mean(Qannual_post1980, na.rm = TRUE), col = "red")
mean(Qannual_pre1980)
## [1] 3.271611
sd(Qannual_pre1980)
## [1] 0.3448535
mean(Qannual_post1980, na.rm = TRUE)
## [1] 4.113711
sd(Qannual_post1980, na.rm = TRUE)
## [1] 0.6309865

Note: the parameter na.rm was set to TRUE for calculating statistics for the post-1980 period. na.rm = TRUE instructs the function to remove NA values before computing the statistic. If we don’t remove the NA values (i.e., the missing part of the record), R won’t calculate the mean or the standard deviation!

The figures show that something probably happened around the year 1980 which has caused the mean and the variance of the streamflow to increase quite substantially. This particular record is impacted by a reservoir upstream, so with some further investigation you might be able to determine whether river regulation caused the apparent change. You could rule out climate factors by looking at rainfall for the catchment to see if a similar pattern emerges, for instance.
